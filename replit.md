# MedRAS

MedRAS is a structured Research Operating System that guides medical and academic researchers from objectives to submission-ready manuscripts.

## Run & Operate

*   `uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload`: Run the FastAPI application.
*   `python -m pytest`: Run tests.
*   **Required Env Vars:** `OPENAI_API_KEY`, `GEMINI_API_KEY`, `COPALEAKS_API_KEY`.

## Stack

*   **Backend:** Python 3.11, FastAPI, Uvicorn
*   **Frontend:** HTML, CSS, JavaScript (no frameworks)
*   **ORM:** _Populate as you build_
*   **Validation:** Pydantic
*   **Build Tool:** _Populate as you build_

## Where things live

*   `/app`: Backend Python source code (API endpoints, services).
*   `/public`: Frontend static files (HTML, CSS, JS) for various modules.
*   `/artifacts/medras`: Files that should not be modified.
*   **DB Schema:** _Populate as you build_
*   **API Contracts:** Implicitly defined by FastAPI Pydantic models.
*   **Theme Files:** `/public/css/style.css`.

## Architecture decisions

*   **No Frontend Frameworks:** Intentional choice for simplicity and direct control.
*   **In-Memory Processing:** All uploaded files are processed in-memory for security and performance.
*   **LLMs for Planning/Writing Only:** LLMs are strictly used for planning and textual tasks; numerical computations are library-backed.
*   **Dual LLM Provider Fallback:** Plagiarism & AI Reduction module uses an `auto` provider strategy, falling back between OpenAI and Google Gemini for resilience.
*   **Smart RAG Infrastructure:** Includes services for domain routing (`rag_router`, format-id map kept in sync with `format.js`), guideline retrieval (`rag_guidelines`), and asynchronous fan-out external database querying (`rag_retriever`). Retriever has 1-hour in-process TTL cache (`CACHE_TTL_S`), parallel PubMed esummary+efetch for full abstracts, and surfaces "requires subscription" messages for Cochrane/CINAHL/IEEE stubs.
*   **Proposal Generator (Step 6 — 7-section drafting):** `app/services/proposal_generator.py` orchestrates router → cached retrieve → single Gemini 2.5 Flash JSON call (12000 max tokens) drafting all seven sections at once: Background, Literature Review, Rationale, Methods, Statistical Plan, Expected Outcomes (all RAG-grounded with `[CITE_n]` tags) plus Ethics (grounded in Helsinki / ICMR 2017 / IEC / ICH-GCP from `rag_guidelines`, refers to standards by name — no `[CITE_n]`). Orphan-citation stripper filters any hallucinated `[CITE_n]` whose index falls outside the retrieved range. System prompt hardened against prompt injection from untrusted abstracts via explicit `=== BEGIN/END UNTRUSTED EVIDENCE ===` framing. Endpoint `POST /api/proposal/generate-rag-sections` (name kept for backward compat). Budget & Timeline are deliberately manual (institutional-rule dependent).
*   **Step 7 — Preview:** `public/proposal-module/preview.html` + `js/preview.js` reads `sessionStorage["medras.proposal.generated"]` set by Step 6 and renders all seven generated sections plus two manual textareas (Budget, Timeline) auto-saved to `sessionStorage["medras.proposal.manual"]`. Cited sources are listed with anchored `[CITE_n]` deep-links from the prose.
*   **Step 8 — Download:** `public/proposal-module/download.html` + `js/download.js` collect title-page metadata (Institution, Committee, study title CAPS, PI/Guide/Co-Guide name+designation+department, year). For Indian-thesis formats only (`md-ms-syn`, `phd-syn`, `inst-diss` — gated by `INDIAN_THESIS_FORMAT_IDS` mirrored on backend as `_INDIAN_THESIS_FORMAT_IDS`) the form also reveals Degree / Specialty / University / Submission date, PI year-of-residency, and optional Phone+Email per person; the cover page then renders an IEC committee header and "Submitted in partial fulfilment…MD/MS in [Specialty] under [University]" line. All fields auto-save to `sessionStorage["medras.proposal.titlepage"]` (shallow-merged on read so newly-added keys don't break old saves). Consent forms use an 11-language checkbox grid (English mandatory + Hindi/Tamil/Telugu/Kannada/Malayalam/Marathi/Bengali/Gujarati/Punjabi/Odia) with a delivery picker (`attached`|`separate`|`both`) saved to `sessionStorage["medras.proposal.consent"]`; payload sends `consent_languages` + `consent_delivery`. POSTs go to `/api/proposal/export/{docx,pdf,zip,plaintext}`. `app/services/proposal_export.py` renders DOCX (python-docx) and PDF (reportlab) with TNR 12pt body / 14pt bold headings, 1.5 line spacing, justified, 1″ margins, page-numbered footers; DOCX uses Word's TOC field, PDF uses a static section list. Consent is produced from a hard-coded English template + Gemini 2.5 Flash translation per language (in-process `_TRANSLATION_CACHE`); when `consent_delivery=="separate"` the inline copy is omitted and `build_zip` adds standalone `__consent.docx` / `__consent.pdf`; `"both"` does both. "Send to Plagiarism Checker" stages text via `sessionStorage["medras.plagiarism.prefill"]` and `plagiarism-module/checker.html` auto-fills `#pm-text` on load.
*   **Step 4 — Outline modal & live sample-size formula:** `public/proposal-module/outline.html` + `js/outline.js` render the section list as clickable rows; each row opens a centred modal popup (overlay + max-width 880, scroll-locked body, Esc/click-overlay/Cancel = discard, Save&Close or Ctrl-Enter = persist) containing a focused textarea, helper buttons (per-section upload via `/api/outline/extract-section`, Sample Size Calculator deep-link, Study Builder soft-handoff to `/#study-builder`), and — only on sections matching `/sample\s*size|statistical\s+(analysis|plan)/i` — a live "Sample-size formula preview" panel. The panel parses values from the textarea (`p=`, `p1=`/`p2=`, `d=`, `Z=` / `α=` (auto-mapped to Z), `power=` (auto-mapped to Z_β), `σ=`, `Δ=` / `μ1=`+`μ2=`, `Sn=`/`Sp=`, `r=`) and substitutes them into one of six formulas (single proportion, two proportions, single mean, two means, sensitivity/specificity, Pearson r via Fisher-z) with a final integer enrolment number (×2 for two-group designs).
*   **Step 5 — References preflight:** `references.html` shows three cards instead of two buttons — "I'll upload reference papers" (`#prop-ref-pre-yes`, primary, existing handler), "Use the Study Builder" (recommended badge, opens `/#study-builder` in a new tab — soft handoff), and "Generate plausible suggestions" (`#prop-ref-pre-no`, fallback with verify-before-citing warning). Original button IDs are preserved so the existing `references.js` wiring is unchanged.
*   **PPT extraction (`outline_extractor.py:_extract_pptx`):** Walks every shape on every slide — text frames, tables, charts (title/axes/series/categories), notes slides, slide-layout placeholders (headers/footers) — and recurses into group shapes. Consecutive duplicates are de-duped before slot-assignment.
*   **Proposal Writing Module State Management:** Uses `sessionStorage` mirrored to `localStorage` for auto-save, mobile responsiveness, and a `beforeunload` guard for in-flight operations.
*   **Plagiarism Pipeline Job Management:** Uses a `JobManager` for daemon thread processing, per-stage timeouts, and circuit breakers, replacing NDJSON streaming with polling.
*   **Plagiarism — RAG Citation Suggestions:** `app/services/citation_suggester.py` extracts up to N citation-worthy claims from a rewritten section via Gemini 2.5 Flash JSON (with anti-hallucination quote-substring check + heuristic fallback when Gemini is unavailable), routes via `rag_router` (topic_hint biases DB selection), then concurrently calls `rag_retriever.retrieve` per claim. Endpoint `POST /api/plagiarism/suggest-citations` (rate-limited 10/min, max 8 claims). Per-section "Suggest real citations" button on `reduce-results.html` renders each claim with up to 3 verified papers (title, authors, year, journal, source database, clickable `https://doi.org/{doi}` link). Empty suggestions are surfaced honestly — never placeholders.

## Product

*   **Helix (Study Builder):** Translates research objectives into methodologies.
*   **Cohort (Sample Size Calculator):** Provides formula-driven sample size estimations.
*   **Sigma (Statistical Analysis Engine):** Guided wizard for data input, variable classification, quality checks, and statistical analysis.
*   **Prologue (Proposal Writing Module):** Guided 8-step intake (Role → Language → Format → Outline → References → Generate → Preview → Download) supporting various academic formats.
*   **Scriptorium (Thesis & Article Writing Module):** Branded "Scriptorium" — the medieval room where scholars wrote manuscripts. Guided 9-station flow (Welcome → Setup → Guidelines → References → Data & Locks → Dashboard → Editor → Compliance → Export) for Indian MD / DNB / PhD theses **and** journal-ready articles (Case Report / Series / Narrative-Systematic-Meta Review / Original Research / Monograph). Welcome page (`index.html`) is a 3-question intake (mode → article type → role) backed by a single source-of-truth catalogue at `js/writing-modes.js` (ROLES + THESIS_TYPES + ARTICLE_TYPES with per-type targets: guideline checklist, word-or-page count, citation min/max, plag cap, AI cap, delivery hint — prices intentionally omitted). Picker persists `setup.mode`, `setup.role`, `setup.article_type`, `setup.target_id` plus a flat `targets` block on `ThesisState` so downstream pages (compliance, export) can read the deliverable contract without re-walking the catalogue. Setup page hydrates an "✦ Article: <Type> · <Role>" banner and adapts fields (hides Guide / Co-Guide and the Indian-format selector for articles; reveals a Co-authors textarea instead). Article-specific chapter spines (CARE / STROBE / PRISMA / CONSORT / COREQ) are stubbed for the next iteration — articles currently fall through to the generic NBEMS spine. Every station carries a consistent 8-step stepbar (Setup → Export) plus a "Station N of 9" eyebrow; the Editor's right-hand pane includes a per-chapter "Attach figure to this chapter" deep-link to `assets.html?chapter=<id>` and forward-nav buttons to Compliance / Export. Compliance lives at its own page (`compliance.html`) — auto-runs on load when chapters have content, gates Export with a green-pass/amber-fail banner. Researcher-in-control: every AI suggestion is an inline track-change requiring an explicit ✓ click; numbers from the researcher's study are locked in yellow and cannot be altered by the AI; AI never writes a sentence without an attached source from the retrieved papers. Persistent IndexedDB reference library across theses ("addictive flexibility"). One canonical Indian-thesis chapter spine + university-rules overlay layer (page count, font, spacing, citation style, ref minimum, declarations) parsed from the uni's guidelines PDF via deterministic regex (no LLM).
*   **Scriptorium — Proforma chapter upload:** The Proforma chapter (after Conclusion in the NBEMS spine) is researcher-provided — not AI-drafted. In the Scriptorium editor, navigating to the Proforma chapter shows: (1) a sky-blue notice banner explaining it is researcher-provided content; (2) a "⬇ Download blank proforma template (DOCX)" link hitting `GET /api/thesis/proforma-template`; (3) an "Upload your filled proforma" helper button that opens the standard import modal adapted for proforma mode (title changed to "Upload your filled proforma", paste textarea hidden, AI Polish button hidden — only "Use as-is" offered). The DOCX template (`app/services/proforma_template.py`) covers 14 sections: Patient Identification · Chief Complaints · HPI · Past History · Personal History · Family History · General Examination · Systemic Examination · Investigations (haematology / biochemistry / radiology / special) · Diagnosis · Treatment · Course in Hospital · Outcome · Follow-up — TNR 12pt, single spacing, A4 with 1″ margins, matching the NBEMS conventional format.
*   **Thesis Writing — Backend (Phase 1+2):** `app/services/thesis_formats.py` defines the 16-chapter NBEMS-style spine (Title → Certificates → Abbrev → Abstract → 8 body chapters → Proforma → Consent → References → Annexures) + DEFAULT_RULES (NBEMS: A4 / TNR 12 / 1.5 spacing / 1″ margins / 80-page max / Vancouver / IEC mandatory / 100 ref minimum). `thesis_guidelines_parser.py` extracts page/font/spacing/margin/citation-style/ref-min/plagiarism-cap from uploaded PDF/DOCX/TXT via regex and merges with defaults. `thesis_reference_library.py` does Crossref DOI verification + distilled RAG search (quality_filter drops stubs/predatory/short titles; composite score = 0.55·overlap + 0.20·recency + 0.15·abstract + 0.10·DOI; per-domain recency window) + heuristic distilled summaries (no LLM dependency). `thesis_section_writer.py` provides `draft_section()` (RAG-grounded fresh draft via Gemini 2.5 Flash JSON, prompt-injection-hardened with `=== BEGIN/END UNTRUSTED EVIDENCE ===` framing, orphan-cite stripper, locked-number enforcement) and `improve_section()` (sentence-level inline-diff suggestions with anti-hallucination substring check + locked-number protection). `thesis_compliance.py` runs 9 pre-flight checks (page count, ref count, verified DOIs, body chapters at length, plagiarism, AI score, declarations, IEC, title-page metadata, asset placement) returning pass/warn/fail items. All endpoints under `/api/thesis/*` (spine, parse-guidelines, references/{verify-dois,extract-dois,search}, draft-section, improve-section, compliance-check). Server stateless — client owns state.
*   **Thesis Writing — Setup-as-AI-grounding:** `public/thesis-module/setup.html` collects not just title-page metadata but a "Research context" block — `aim`, `primary_objective`, `hypothesis`, `population`, `setting`, `study_type`, `study_period` — which the editor passes to `/api/thesis/draft-section` as `extra_context`. An auto-detected "Import from my MedRAS Proposal" bar (top of Setup) reads `sessionStorage["medras.proposal.intake"|"generated"|"titlepage"|"topic"]` and fills any blank fields; the proposal's generated `background` section is stashed at `setup._proposal_background` (truncated to 4000 chars in editor) for AI grounding only. Editor's `handleHelper('draft'|'improve')` hard-gates on `setup.title` — without a real title it refuses client-side and bounces the researcher to Setup, because falling back to the chapter label (e.g. "Chapter I — Introduction") makes RAG return junk and the backend rightly rejects the under-cited Gemini draft with a 400.
*   **Thesis Writing — Frontend:** Emerald/parchment theme (visually distinct from Proposal's navy) at `/public/thesis-module/`. State management in `js/thesis-state.js`: sessionStorage mirrored to localStorage for the active thesis + IndexedDB store `medras-thesis-lib` for the persistent cross-thesis reference library (survives all browser refreshes; "import from saved library" button on References page). Inline track-changes in `js/track-changes.js`: non-overlapping placement algorithm renders each suggestion's `original` with strikethrough + `suggested` in green inline with ✓/✗ buttons; yellow-locked-numbers wrapper auto-decorates every locked value across both write and review modes. Three-pane editor (left: chapter outline + side locks + side refs; centre: contentEditable canvas with Write/Review toggle; right: helper strip + AI suggestions list with per-card accept/reject). Helper strip per chapter is driven by the `helpers[]` array in CHAPTER_SPINE — deep-links to `/sample-size.html`, `/analysis.html`, `/plagiarism-module/checker.html` (with prefill via `sessionStorage["medras.plagiarism.prefill"]`), Study Builder. Pages: index (4-mode welcome), setup, guidelines (uploader + NBEMS-defaults toggle, evidence-list of detected rules), references (DOI add + RAG search + import-from-saved), data (locked-numbers UI with quick-add chips), dashboard (per-chapter traffic-light grid with summary cards + run-compliance), editor.
*   **Novus (Plagiarism & AI Reduction):** Offers originality scoring, AI likelihood detection, and a 3-stage rewrite pipeline.
*   **Folio (Document Management — Module 07):** Import or create thesis/manuscript documents. Entry page at `public/folio-module/index.html` (start new blank doc or upload DOCX). Editor at `public/folio-module/editor.html` — 3-pane layout: section outline (left, drag-to-reorder), paper-like document canvas (centre, contenteditable paragraphs), reference panel (right, style tabs). State key: `sessionStorage["folio.doc"]` (document model JSON). Sigma handoff via `sessionStorage["folio.import.from_sigma"]` triggered by the "Take to Folio" button in `public/js/analysis.js:bindResults()`. Backend: `app/api/folio.py` (prefix `/folio` under the `/api` mount): `POST /api/folio/import-docx` (DOCX → structured JSON), `POST /api/folio/format-references` (reformat reference list in vancouver/apa/harvard), `POST /api/folio/parse-feedback` (Gemini 2.5 Flash parses supervisor plain-English feedback into structured operations), `POST /api/folio/export-docx` (document model → downloadable DOCX, TNR 12pt, 1.5 spacing, 1″ margins). Smart tracking: `FolioCore.deleteRef()` renumbers all `[N]` in-text citations; `renumberTables/Figures()` renumber `Table N` / `Figure N` patterns across all paragraphs; Ctrl+Z restores one localStorage snapshot. Citation styles: clicking Vancouver/APA/Harvard tab calls `/api/folio/format-references` and re-renders the panel. JS modules: `public/folio-module/js/folio-core.js` (pure state/data — no DOM), `public/folio-module/js/folio-editor.js` (all DOM rendering and event binding).

## Gotchas

*   Uploaded files are subject to 100 MB byte cap and 200-page PDF cap.
*   Plagiarism module's rewrite pipeline automatically skips References/Bibliography sections.
*   Quota exhaustion for LLM providers will lead to temporary service unavailability for plagiarism rewrites.
