CHANGE 1 — WIRE THE WIZARD TO THE 
VARIABLE ASSIGNMENT
════════════════════════════════════

When the wizard completes and the user 
uploads their file, run this logic 
immediately after the file loads:

Take the outcome description the user 
typed in Q3 of the wizard.
Search all column names in the uploaded 
file for the closest match.
Use case-insensitive partial matching.
Example: user typed "time to union" 
→ matches column "Time to Union (Weeks)"

Rules for matching:
  Never match an ID column even if the 
  name is similar.
  Never match a Date column.
  If exact match found: use it.
  If partial match found: use it.
  If no match found: flag it 
  (see Change 3 below).

Take the objective text the user typed 
in Q2.
Scan for comparison keywords: 
  "between", "compare", "across", 
  "by", "among", "in different", 
  "vs", "versus", "groups"
The word or phrase after these keywords 
is the likely grouping variable.
Find the closest matching column name.
Set that as the grouping variable.

Save both into the session:
  session.outcome_variable = matched column
  session.grouping_variable = matched column
