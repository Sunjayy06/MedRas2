─────────────────────────────────
RULE 1 — SCALE vs ORDINAL
─────────────────────────────────

A variable is SCALE (continuous or discrete) 
if it measures a quantity with real numeric 
meaning — regardless of how many unique 
values it has.

Examples that MUST be classified as SCALE:
  Age, Weight, Height, BMI
  Haemoglobin, Blood pressure, Temperature
  Creatinine, eGFR, HbA1c
  Time to union (weeks or days)
  Operating time (minutes)
  Duration of surgery
  Harris Hip Score (0 to 100)
  Radiological Union Score (numeric)
  Any score that produces a number 
  on a continuous or count scale

A variable is ORDINAL only when:
  The values have a natural rank order
  AND the gap between values is not 
  equal or not meaningful
  Examples: NYHA class (I, II, III, IV),
  Pain grade (mild, moderate, severe),
  Education level (primary, secondary, 
  tertiary), Tumour stage (1, 2, 3, 4)

Harris Hip Score, Radiological Union Score, 
VAS Score, and similar clinical scoring 
systems are SCALE by default, not ordinal.
They produce a number. Mean and SD apply.
Do not classify them as ordinal just because 
they have a limited number of unique values.

The rule for ordinal is NOT based on 
how many unique values a column has.
It is based on what the variable measures.

Remove any logic that says 
"if unique values < 10 then ordinal".
Replace with column name pattern matching:
  If column name contains: score, index, 
  scale, VAS, NRS, HHS, Harris, Union, 
  time, duration, age, weight, height, 
  BMI, pressure, rate, level, count, days,
  weeks, months, minutes, hours, length, 
  distance, volume, dose — classify as SCALE.

STEP 5 — NORMALITY CHECK
─────────────────────────────────

For every variable classified as Scale, 
run a normality test automatically.

Which test to use:
  n below 50 → Shapiro-Wilk
  n 50 or more → Kolmogorov-Smirnov
  n above 2000 → skip test, treat 
  as non-normal automatically

Also check:
  Skewness: if absolute value above 2 
  → non-normal
  Kurtosis: if absolute value above 7 
  → non-normal

Decision rule:
  Normal if: test p above 0.05 AND 
  skewness below 2 AND kurtosis below 7
  Non-normal if: any one of those fails
  Borderline if: p between 0.01 and 0.05 
  → run both parametric and non-parametric 
  and show both results

If non-normal: try log transformation.
  If log transformation makes it normal,
  use parametric tests on log-transformed 
  data and note this in the report.
  If transformation does not work, 
  use non-parametric.

Show results in a table:
  Column | Test used | p-value |

    Red = Non-normal
