PART 2 — PRACTICE DATA WIZARD (6 STEPS)
--------------------------------------------------

Create a step-by-step wizard (same UI style as existing flow)

Step 1:
Ask: Study objective (text area, required)

Step 2:
Ask: Outcome variable (text, required)

Step 3:
Ask: Variables (multi-line text)
Example:
Age
Gender
HbA1c
BMI

After user enters:
- Detect type automatically:
  Age → Scale
  Gender → Nominal
  HbA1c → Scale
  Status/Outcome → Binary
- Show table:
  Variable | Type | Dropdown to edit

Step 4:
Ask: Sample size
- Default: 60
- Range: 20–500

Step 5:
Show dynamic inputs based on variables:
- Gender → Male %
- Disease → % with condition
- Age → Min / Max
- Outcome → % good outcome
- Optional notes field

Step 6:
Ask expected result (optional)
Example: "HbA1c higher in diabetics"

Next → Show summary screen

--------------------------------------------------

- Add 5% missing values

Also create function:

- Row 1: RED disclaimer (practice data)

PART 4 — BACKEND ROUTES
--------------------------------------------------

Add these APIs:

1. POST /api/generate-practice-data
- Generate dataset
- Save temporary Excel file
- Return file for download
- Store session info

2. POST /api/use-practice-for-analysis
- Load generated dataset
- Push into analysis pipeline

3. GET /export/<id>/excel-original
- Download original dataset

4. GET /export/<id>/excel-cleaned
- Download cleaned dataset
