Add a smart 3-step intake flow to the plagiarism module. This appears when the user clicks "Reduce Plagiarism" before any file upload happens. Do not change any other part of the Medras application.

STEP 1 — Ask if they have already checked with plagiarism software:

Show a clean card with the heading "Before we start — have you already checked your document?"

Show two large option buttons:

Button A: "Yes, I have a plagiarism report"
Description under button: "You have already run your document through Turnitin, Drillbit, Aarjav, iThenticate, Urkund or any other plagiarism checker and have the report"
Button B: "No, I want to process my original document directly"
Description under button: "You have not checked yet or do not have a report — we will process your full document"

Below the two buttons show a small text line: "Supported plagiarism software reports: Turnitin, Drillbit, Aarjav, iThenticate, Urkund, PlagScan, Copyleaks, or any PDF/DOCX report"

STEP 2A — If user clicks "Yes I have a plagiarism report":

- Drag and drop zone for PDF or DOCX

Right column headed "Step 2 — Upload your PLAGIARISM REPORT":
- Drag and drop zone for PDF, DOCX or TXT

Turnitin, Drillbit, Aarjav, iThenticate, Urkund, Copyleaks, PlagScan, Other

STEP 2B — If user clicks "No, process directly":

Show a single upload zone:
- Drag and drop for PDF or DOCX
- Label: "Upload your document"
- Helper text: "We will analyse your full document and rewrite all sections that need improvement"
- Green "Analyse My Document" button

STEP 3 — Backend processing for Path A (with plagiarism report):

Add a new backend function in the plagiarism service that:
- Receives both the original document and the plagiarism report
- Extracts text from the original document as already done
- Extracts text from the plagiarism report using the same pdf-parse or mammoth extraction
- Scans the report text to identify flagged sections by looking for these patterns:
  * Percentage similarity values like "Introduction: 34%" or "34% similarity"
  * Section names followed by highlight descriptions
  * Phrases like "high similarity", "match found", "plagiarised", "flagged"
  * Any section name from the known list (Abstract, Introduction, Methods etc) that appears near a percentage above 10%
- Build a flagged sections map: { "Introduction": { similarityPercent: 34, flagged: true }, "Discussion": { similarityPercent: 67, flagged: true }, "Methods": { similarityPercent: 8, flagged: false } }
- Sections with similarity above 10% are marked as priority rewrite
- Sections with similarity below 10% are marked as low priority — rewrite lightly or skip
- Pass this flagged map to the rewriting pipeline

REWRITING BEHAVIOUR based on plagiarism report:

In the rewriting pipeline, use the flagged sections map to control rewriting intensity:

RED priority sections (similarity above 30%): Run all 3 rewriting stages with maximum intensity. Use the most aggressive restructuring techniques.

ORANGE priority sections (similarity 15 to 30%): Run all 3 stages with moderate intensity.

YELLOW priority sections (similarity 10 to 15%): Run stage 1 and stage 2 only. Light restructuring.

GREEN sections (similarity below 10%): Skip rewriting entirely. Return original text unchanged. Show a green tag "Already within acceptable limits — not rewritten"

References and Bibliography: Always skip regardless of similarity score.

DISPLAY on results page:

For Path A (with report) show at the top:

- Each section card should show the original similarity percentage from the report in a badge like "Was 34% similar" next to the section heading
- After rewriting show: "Targeted rewriting applied to high-similarity sections only"

For Path B (without report) show:
- "Full document processed — all sections analysed and rewritten for originality"

NAVIGATION:
