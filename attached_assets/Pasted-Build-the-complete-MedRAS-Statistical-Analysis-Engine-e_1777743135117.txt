PART 2 — VARIABLE CLASSIFICATION ENGINE
(Fix this once, permanently)
════════════════════════════════════

The classification engine runs when a file 
is uploaded. It must never misclassify 
a clinical score as Ordinal when it is Scale.

RULE 1 — Scale classification by column name:
Any column whose name contains any of these 
words must be classified as Scale automatically:
  score, index, scale, VAS, NRS, HHS, harris,
  union, time, duration, age, weight, height,
  bmi, pressure, rate, level, count, days,
  weeks, months, minutes, hours, length,
  distance, volume, dose, hb, haemoglobin,
  creatinine, egfr, hba1c, glucose, sodium,
  potassium, calcium, temperature, spo2,
  pulse, bp, systolic, diastolic, radiological,
  harris hip, operating, surgery

This rule overrides any logic based on 
unique value count. Do not use "fewer than 
10 unique values means ordinal". Remove that 
rule permanently.

RULE 2 — Ordinal only when explicitly ordered:
A variable is Ordinal only if its values 
contain words like:
  mild, moderate, severe
  low, medium, high
  grade 1, grade 2, grade 3
  stage 1, stage 2
  class I, class II, class III
  NYHA, TNM, WHO grade
  never, sometimes, always
  strongly disagree, disagree, agree

RULE 3 — String mixed variables:
When a column contains values like 
"Grade 4", "2 mm", "Score 7", "Level 3":
  Strip all non-numeric text
  Keep only the number
  Classify as Scale
  Show user: "We removed the text prefix 
  from [column]. Values converted to numbers. 
  Original column kept as [column]_original."

RULE 4 — Discrete vs continuous:
Both are treated as Scale for all analyses.
Discrete = whole numbers only (count data)
Continuous = can have decimals
Both use mean, SD, and the same tests.
Show sub-type as information only.

RULE 5 — ID columns:
Never usable as outcome or grouping variable.
Grey them out in all dropdowns.
Block if somehow selected.

PART 4 — STATISTICAL TEST SELECTION ENGINE
(The brain of the product)
════════════════════════════════════

This engine runs after normality is confirmed.
It takes the variable types, normality results,
and study structure as inputs.
It outputs a list of tests to run.
It uses rules only — no AI, no guessing.

THE DECISION TREE:

ALWAYS RUN FIRST:
  Descriptive statistics for all variables
  Cannot be removed or skipped

COMPARISON TESTS — select based on:
  How many groups (2 groups, 3+ groups, paired)
  Variable type (continuous, categorical)
  Normality result (normal, non-normal)

  2 independent groups, continuous outcome:
    Normal: Independent t-test
      Check Levene test for equal variance
      If Levene p below 0.05: use Welch t-test
    Non-normal: Mann-Whitney U

  2 paired groups (before/after, matched):
    Normal: Paired t-test
    Non-normal: Wilcoxon signed-rank

  3 or more groups, continuous outcome:
    Normal: One-way ANOVA
      If Levene p below 0.05: 
      Welch ANOVA (Brown-Forsythe)
      Post-hoc: Tukey if equal variance
      Post-hoc: Games-Howell if unequal variance
    Non-normal: Kruskal-Wallis
      Post-hoc: Dunn with Bonferroni correction

  Repeated measures, 3 or more timepoints:
    Normal: Repeated Measures ANOVA
      Check Mauchly sphericity
      Apply Greenhouse-Geisser if violated
    Non-normal: Friedman
      Post-hoc: Wilcoxon with Bonferroni

  Mixed design (groups x timepoints):
    Normal: Mixed ANOVA
    Non-normal: use Mixed Linear Model

  2 groups, categorical outcome:
    Expected cell count 5 or above: Chi-square
    Any cell below 5: Fisher exact
  
  3 or more groups, categorical outcome:
    Chi-square
    Fisher-Freeman-Halton if cells below 5
  
  Paired categorical (before/after binary):
    McNemar test

CORRELATION:
  Both continuous, both normal: Pearson
  Either non-normal or ordinal: Spearman
  Both binary: Phi coefficient
  Both categorical: Cramers V
  Multiple variables: correlation matrix 
  with heatmap

REGRESSION — select based on outcome type:
  Continuous outcome, one predictor:
    Simple linear regression
  Continuous outcome, multiple predictors:
    Multiple linear regression
    Run univariate first
    Include variables with p below 0.25 
    in multivariate
    Check VIF for multicollinearity
    Warn if VIF above 10
  Binary outcome (Yes/No, 0/1):
    Binary logistic regression
    Run univariate then multivariate
    Report crude and adjusted OR with 95% CI
    Generate forest plot
    Run Hosmer-Lemeshow goodness of fit
  Ordinal outcome:
    Ordinal logistic regression
  Count outcome:
    Poisson regression
    Check overdispersion
    If overdispersed: negative binomial
  Time to event outcome:
    Cox proportional hazards regression
    Check Schoenfeld residuals for PH assumption
    If violated: stratified Cox or time-varying

SURVIVAL — trigger when:
  Time column + Event column (0/1) detected
  OR objective text contains: survival, 
  mortality, time to, death, recurrence
  
  Always run:
    Kaplan-Meier per group
    Log-rank test
    At-risk table below KM curve
  
  If predictors present:
    Cox proportional hazards regression

DIAGNOSTIC ACCURACY — trigger when:
  Binary outcome + test result column
  OR objective contains: sensitivity, 
  specificity, ROC, diagnostic, screening
  
  Calculate all from 2x2 table:
    Sensitivity with 95% CI
    Specificity with 95% CI
    PPV with 95% CI
    NPV with 95% CI
    Positive Likelihood Ratio
    Negative Likelihood Ratio
    Diagnostic Odds Ratio
    Accuracy
  
  ROC curve with AUC and 95% CI
  Youden index for optimal cutoff

RELIABILITY — trigger when:
  Two rater columns detected
  OR objective contains: agreement, 
  reliability, kappa, ICC, inter-rater
  
  Categorical: Cohen Kappa with interpretation
  Ordinal: Weighted Kappa
  3 or more raters: Fleiss Kappa
  Continuous: ICC + Bland-Altman plot

EFFECT SIZE — add to every test result:
  t-test: Cohen's d
  ANOVA: eta-squared and omega-squared
  Mann-Whitney: r = Z divided by sqrt(N)
  Kruskal-Wallis: eta-squared H
  Chi-square 2x2: Phi coefficient
  Chi-square larger: Cramers V
  Correlation: r or rho is the effect size
  Linear regression: R-squared and f-squared
  Logistic regression: OR with 95% CI
  Cox: HR with 95% CI
  Kappa: kappa value itself
  AUC: AUC value itself

MULTIPLE TESTING CORRECTION:
  When more than 3 outcomes are tested:
    Apply Bonferroni for small number of tests
    Apply Benjamini-Hochberg FDR for many tests
  Always report both uncorrected 
  and corrected p-value

PART 6 — STATISTICAL ACCURACY CONTROLS
(Ensuring results match SPSS and R)
════════════════════════════════════

CONTROL 1 — Library version pinning:
  Pin all library versions in requirements.txt
  scipy >= 1.11.0
  statsmodels >= 0.14.0
  pingouin >= 0.5.3
  lifelines >= 0.27.0
  This ensures reproducible results.

CONTROL 2 — Two-tailed tests by default:
  All hypothesis tests use two-tailed 
  alpha = 0.05 unless otherwise specified.
  Always report exact p-value to 3 decimal places.
  Report p < 0.001 when p is very small.

CONTROL 3 — Exact formula verification:

  For independent t-test:
    Use scipy.stats.ttest_ind
    Check Levene test first: scipy.stats.levene
    If Levene p below 0.05: 
    use equal_var=False (Welch)
    This matches SPSS exactly.

  For Mann-Whitney U:
    Use scipy.stats.mannwhitneyu
    alternative='two-sided'
    This matches SPSS exactly.

  For ANOVA:
    Use scipy.stats.f_oneway
    Post-hoc Tukey: 
    statsmodels.stats.multicomp.pairwise_tukeyhsd
    Post-hoc Games-Howell: 
    pingouin.pairwise_gameshowell
    These match SPSS exactly.

  For Kruskal-Wallis:
    Use scipy.stats.kruskal
    Post-hoc Dunn: 
    scikit_posthocs.posthoc_dunn
    with p_adjust='bonferroni'
    This matches SPSS exactly.

  For Chi-square:
    Use scipy.stats.chi2_contingency
    correction=False for tables larger than 2x2
    correction=True only for 2x2 with small n
    This matches SPSS exactly.

  For Fisher exact:
    Use scipy.stats.fisher_exact for 2x2
    For larger tables: 
    scipy.stats.chi2_contingency with 
    correction note in output
    This matches SPSS exactly.

  For Pearson correlation:
    Use scipy.stats.pearsonr
    This matches SPSS exactly.

  For Spearman correlation:
    Use scipy.stats.spearmanr
    This matches SPSS exactly.

  For logistic regression:
    Use statsmodels.formula.api.logit
    Report Nagelkerke R-squared
    Run Hosmer-Lemeshow test
    These match SPSS exactly.

  For linear regression:
    Use statsmodels.formula.api.ols
    Report adjusted R-squared
    Run Breusch-Pagan for homoscedasticity
    Run Durbin-Watson for independence
    These match SPSS exactly.

  For Kaplan-Meier:
    Use lifelines.KaplanMeierFitter
    Log-rank: lifelines.statistics.logrank_test
    These match SPSS exactly.

  For Cox regression:
    Use lifelines.CoxPHFitter
    Test PH assumption using 
    Schoenfeld residuals
    This matches SPSS exactly.

  For Repeated Measures ANOVA:
    Use pingouin.rm_anova
    Check sphericity with Mauchly test
    Apply Greenhouse-Geisser if violated
    This matches SPSS exactly.

  For ICC:
    Use pingouin.intraclass_corr
    Specify ICC type explicitly
    This matches SPSS exactly.

  For Shapiro-Wilk:
    Use scipy.stats.shapiro
    This matches SPSS exactly.

CONTROL 4 — Numerical precision:
  All intermediate calculations use 
  float64 precision.
  Round only at display time.
  Never round intermediate values.

CONTROL 5 — Missing data handling in tests:
  Always use listwise deletion for 
  the specific variables in each test.
  Report exact n used in each test.
  This matches SPSS default behaviour.

CONTROL 6 — Effect size calculations:
  Cohen's d = mean difference divided 
  by pooled SD
  Eta-squared = SS_between divided by SS_total
  Omega-squared = (SS_between minus 
  df_between times MS_error) divided by 
  (SS_total plus MS_error)
  r for Mann-Whitney = Z divided by sqrt(N)
  Cramers V = sqrt(chi2 divided by 
  (n times min(rows-1, cols-1)))
  All these match SPSS and R exactly.
