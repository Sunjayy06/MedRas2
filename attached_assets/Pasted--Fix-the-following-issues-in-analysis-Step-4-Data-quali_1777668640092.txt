FIX 2 — QUALITY SCORE CALCULATED CORRECTLY
════════════════════════════════════

The quality score must NOT show 100/100 just 
because there are no outliers or duplicates.

Recalculate the quality score to include 
missing data:

  Score = 100 minus deductions:
  - Each variable with > 50% missing: -10 points
  - Each variable with 20–50% missing: -5 points
  - Each variable with 5–20% missing: -2 points
  - Each outlier flagged: -2 points
  - Each duplicate row: -3 points
  - Each consistency error: -3 points
  - Minimum score: 0

Display the score with colour:
  90–100: green (#3B6D11)
  70–89:  amber (#854F0B)
  Below 70: red (#A32D2D)

Show a small note below the score:
  "Score accounts for missingness, 
   outliers, duplicates, and consistency."

FIX 4 — CATEGORICAL CONSISTENCY CHECKS
════════════════════════════════════

Section C "Data consistency errors" is currently 
only checking for logical date/age errors.
Add these additional checks:

CHECK 1 — Case inconsistency in categorical columns:
  For every Nominal column, get all unique values.
  If the same text appears with different 
  capitalisation (e.g. "neck fracture", 
  "NECK FRACTURE", "Neck Fracture"):
    Flag as: Issue = "Inconsistent capitalisation"
    What to do options: 
      "Standardise to Title Case"
      "Standardise to UPPER CASE"  
      "Keep as is"

CHECK 2 — Near-duplicate text values:
  If two unique values differ only by:
    Extra space, trailing space, double space
    Common abbreviation (e.g. "lt" vs "left")
    Single character typo
  Flag as: Issue = "Possible duplicate label"
  What to do: "Merge these values" / "Keep separate"

CHECK 3 — Numeric values stored as text:
  If a column classified as Nominal contains 
  values like "4", "7", "2" or "Grade 4", 
  "Level 3", "Score 7":
    Flag as: Issue = "Numeric text — may be 
    a score variable"
    What to do: "Convert to numeric" / "Keep as text"

These checks run on all columns automatically 
when Step 4 loads.
Results go into Section C table.
