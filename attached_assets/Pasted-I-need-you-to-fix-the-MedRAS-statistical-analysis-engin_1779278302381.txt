SECOND THING — Use AI to bridge the gap 
between what users say and what is in 
their Excel file

This is the most important part. After 
the user fills in the starting screen and 
uploads their Excel file, the system 
needs to figure out which column in the 
Excel file is their outcome variable and 
what kind of study they are describing.

People do not always use precise 
statistical language. Someone studying 
ER expression might write association 
study or expression pattern or 
immunohistochemical assessment. Someone 
with a P/N column might describe it as 
ER positivity or receptor status or 
expression outcome. The system needs 
to handle all of these naturally.

Here is how to do it. First try simple 
text matching. Take what the user wrote 
in the outcome field and compare it to 
the column names in their Excel file 
using case insensitive partial matching. 
If P/N was typed and there is a column 
called P/N then use it. If HbA1c was 
typed and there is a column with that 
name in any capitalisation then use it. 
This handles the straightforward cases 
without needing AI.

If the simple matching does not find 
a clear match then use Gemini. Send 
Gemini two things. The complete list of 
column names from the uploaded Excel 
file. And the full description the user 
wrote about their study plus what they 
typed in the outcome field. Ask Gemini 
to read both and tell you which single 
column from the list is most likely 
the outcome variable the user is 
describing. Gemini returns one column 
name. Python uses that column name as 
the outcome. Gemini does nothing else. 
No statistics, no test selection, just 
identifying the correct column name.

Also use Gemini to clarify the study 
type when the user's language is 
ambiguous. For example if someone writes 
association of ER expression with 
clinical parameters that sounds like 
it could be descriptive or correlation. 
Send the description to Gemini and ask 
it to classify the study as one of five 
types: correlation, comparison, 
diagnostic, survival, or descriptive. 
Gemini returns one word. Python uses 
that word to decide which analysis 
pathway to follow.

After both the outcome column and study 
type are identified, show the user a 
confirmation card. Say something like 
we understood this as a correlation

variable as P/N. All other columns will

Is this correct? Give them a yes button

All remaining columns that are not ID

FOURTH THING — Clean messy data 
automatically and never exclude 
clinical variables

Before classifying any variable, 
run automatic cleaning on every 
column. The goal is to include 
everything, not to exclude things 
because they look messy.

First identify true ID columns. A 
column is only an ID column if its 
name contains words like serial number, 
accession, registration, patient id, 
or reg no. Or if its values follow a 
pattern like 1234/2025 or sequential 
numbers. Do not mark clinical variables 
as ID just because they have text in them.

Second standardise yes and no columns. 
Look at every column's values. If a 
column has a mixture of words that all 
mean yes such as yes, present, positive, 
found, detected, involved, angio yes, 
lymphatic yes, angio greater than four 
vessels, map all of them to Yes. If it 
has words that mean no such as no, 
absent, negative, not found, not 
assigned, not involved, none, map all 
of them to No. Keep the original values 
in a backup column. Write what was done 
in the cleaning log.

Third extract numbers from columns that 
have numbers mixed with text. For 
example values like 2.4 cm or 0.5 cm T1a 
or Right-2.8 contain a usable number. 
Try to pull out the number from each 
value. If more than sixty percent of 
the values in a column successfully 
yield a clean number, replace the column 
with just the numbers, classify it as 
a continuous variable, and keep the 
original in a backup column.

Fourth remove any existing logic that 
excludes a column because it has messy 
text, many categories, or long string 
values. The only valid reasons to exclude 
a variable are that it is a true ID 
column or the user explicitly chose to 
exclude it in the missing data step.

After cleaning show a blue summary card 
at the top of Step 3 listing every 
cleaning action taken. The user must 
click confirm before seeing the variable 
table. If Gemini is available, add a 
plain English sentence per action 
explaining why it was done.

FIFTH THING — Choose the correct 
statistical tests

After the study type is confirmed and 
the outcome variable is identified, 
Python decides which tests to run 
based purely on the variable types 
and the study type. No AI involvement 
in test selection.

If the study type is correlation, run 
a separate analysis between the outcome 
variable and every other variable in 
the dataset. Skip only ID columns and 
columns the user chose to exclude.

For each pair decide the test like this. 
If the outcome is binary or categorical 
and the other variable is continuous, 
run Mann-Whitney U and show a box plot. 
If the outcome is binary or categorical 
and the other variable is also 
categorical, calculate expected counts 
first. If any expected count is below 
five run Fisher exact test. Otherwise 
run Chi-square. Show a stacked bar chart. 
If the outcome is binary or categorical 
and the other variable is ordinal, run 
Kruskal-Wallis and show a box plot. 
If both variables are continuous, run 
Pearson or Spearman based on normality 
and show a scatter plot.

Always run descriptive statistics for 
every variable first before any 
hypothesis testing. Always create a 
summary table of all pairwise results 
at the end sorted by p-value.

If the study type is comparison, use 
the existing test selection logic with 
no changes. If the study type is 
diagnostic, use the existing diagnostic 
accuracy flow. If survival, use the 
existing survival analysis flow. If 
descriptive, run descriptive statistics 
and graphs only with no hypothesis tests.
